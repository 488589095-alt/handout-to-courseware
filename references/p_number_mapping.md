# P 号映射规则（双 mode 检测 + 严格 P 号映射）

## 为什么这是 v1 的核心改进

v0 让 LLM 自由判断活动展开几页 PPT，效果不稳定：教学设计明明写"对应课件 P9-P13"（5 页），LLM 经常压成 1 页。

v1 引入**双 mode 检测**：教学设计稿带 PN 标注 → 严格映射；不带 → 才让 LLM 推断。

实际跑过的 4 个 case（分解质因数、牛吃草、必胜策略、递推计数），高途的标准教学设计稿**都带 PN 标注**——严格模式是常态。

## 双 mode 检测

```python
def detect_mode(teaching_design_raw_text: str) -> str:
    """检测教学设计是否带'对应课件 PN'标注"""
    if "对应课件P" in teaching_design_raw_text or "对应课件 P" in teaching_design_raw_text:
        return "strict_pn"     # 严格 P 号映射
    return "llm_infer"          # LLM 推断（fallback）
```

扫描 `teaching_design.json` 里 segments[5]（"六、详细教学过程设计"）的 raw_text 即可。

## 严格 P 号模式规则

### 规则 1：slide_num 严格等于 P 号

教学设计明确标的 P 号必须 1:1 映射：

```
教学设计原文                              → slide_structure
─────────────────────────────────────────────────────────
6.2.1 类型一：拿走最后一枚赢（对应课件P6-P11）
  例1（对应课件P11）                       → P11: example_1
6.2.2 类型二：拿走最后一枚输（对应课件P12-P20）
  核心转化思想（对应课件P12）              → P12: type2_concept
  例2（对应课件P20）                       → P20: example_2
  练1（对应课件P19）                       → P19: practice_1
练2（对应课件P21）                         → P21: practice_2
6.2.3 笔记总结一（对应课件P22）            → P22: notes_1
```

### 规则 2：P 号范围必须拆成对应张数

```
教学设计：6.2.1 问题一：对折剪绳问题（对应课件P9-P13）
         = 5 张 slide（P9, P10, P11, P12, P13），不是 1 张！
```

5 页内容拆分参考（基于教学设计原文的自然分段）：

```
P9:  问题标题 + 核心探究目标
P10: 引导填表（直接放表格）
P11: 引导发现 + 核心公式
P12: 例1（教学设计明确"例1 P12"）
P13: 练1（教学设计明确"练1 P13"）
```

### 规则 3：教学设计的子标注优先于父标注

教学设计常出现父+子 PN 嵌套，子标注更细，优先采纳：

```
6.4.1 例5：小蚂蚁走房间（对应课件P34-P35）  ← 父：例5 占 P34-P35
  问题（对应课件P35）                       ← 子：问题在 P35
```

正确映射：
- P34 = 例5引入/题号页（轻量引入）
- P35 = 完整问题（含题干 + 解答）

**v1 一开始就是因为忽略子标注而踩坑**：把"问题"放在 P34，"解答"放在 P35，违反了 `问题（对应课件P35）`。

### 规则 4：教学设计的"表 2 模块范围"决定模块封面位置

教学设计的"二、教材分析 → 2.1 课件内容结构"通常含一张总表：

```
| 模块 | 页码 | 核心内容 | 难度 | 教学目标 |
| 模块一：取棋子问题 | P5-P22 | ... |
| 模块二：对称性 | P23-P29 | ... |
| 模块三：棋盘问题 | P30-P37 | ... |
```

由此推断：
- P5 = 模块一封面
- P23 = 模块二封面
- P30 = 模块三封面

### 规则 5：未明确 PN 的位置用合理推断

P 号范围内**没明确标注的页**，按教学逻辑填充：

```
模块二 P23-P29 = 7 页，明确标注的只有 5 个：
  P25 两堆一样多 / P26 两堆不一样多 / P27 例3 / P28 练3 / P29 笔记二

未明确：P23, P24
合理推断：
  P23 = 模块二封面
  P24 = 模块二情境引入（"两堆问题"故事开场）
```

### 规则 6：教学设计文档顺序 ≠ P 号顺序

教学设计稿的章节顺序和 PPT 页号顺序可能不一致（设计师把"笔记总结三"放在"例 5"页号之前）。**永远按 P 号顺序生成 slide**，不按教学设计的章节顺序：

```
教学设计章节顺序：    6.4.2 例4 → 练4 → 6.4.3 拓展 → 例5 → 6.4.4 笔记总结三
教学设计 P 号：       P34   P35   P39        P38     P37

PPT 实际顺序（按 P 号升序）：
  P34 例4 → P35 练4 → P36 ? → P37 笔记总结三 → P38 例5 → P39 拓展
```

注意 P36 教学设计未明确，可推断为"练4 答案讲解"或"棋盘问题深化"。

## 严格 P 号模式 vs LLM 推断模式 - 选择决策树

```
教学设计 segment 6 含"对应课件P" 字符串？
├── 是 → 严格 P 号模式
│       1. 抽出所有 PN 标注，建立 P 号 → 内容的精确映射
│       2. 看教学设计表 2 决定各模块封面 P 号
│       3. 未明确 P 号的范围内位置用合理推断填充
│       4. 按 P 号升序生成 slide_structure.slides
│
└── 否 → LLM 推断模式（参考 activity_to_layout_rules.md）
        1. 按 activity 类型映射主版式
        2. 按 activity 内容数量决定页数
```

## 实际跑过的 case：4 份教学设计的 PN 覆盖率

| 讲次 | 模块封面 | 教学内容 PN | 笔记 PN | 总结 PN | 课后 PN | 总覆盖 |
|---|---|---|---|---|---|---|
| 牛吃草 | P5/P17/P26 | ✅ 完整 | ✅ 完整 | ✅ P41 | ✅ P42 | ~95% |
| 必胜策略 | P5/P23/P30 | ✅ 完整 | ✅ P22/P29/P37 | ✅ P41 | ✅ P42-P43 | ~90% |
| 递推计数 | P5/P17 | ✅ 完整 | ✅ P16/P33 | ✅ P38 | ✅ P39-P40 | ~90% |
| 分解质因数 | 无 | LLM 推断 | LLM 推断 | LLM 推断 | LLM 推断 | 0%（LLM 推断模式）|

## 严格模式下生成 slide_structure 的标准做法

伪代码：

```python
1. 从 teaching_design.json 的 segment 6 raw_text 抽出所有 "（对应课件P\d+(-P\d+)?）" 标注
2. 解析为 {章节名: P 号或 P 号范围}
3. 排序，按 P 号升序排列 slide
4. 每个 slide 的 source、placeholders、add_text_boxes 按以下规则填：
   - 模块封面：source=template_idx idx=4，placeholders={"10": "0N.", "11": "模块N\n..."}
   - 例题：source=template_idx idx={6/10/13/16/19}，placeholders={"11": "例 N", "16": <题干>}
   - 练习：source=template_idx idx={7/11/14/17/20}，placeholders={"11": "练 N", "10": <题干+参考答案>}
   - 笔记：source=template_idx idx=8 或 duplicate idx=8，placeholders={"11": <小节标题>}, add_text_boxes=[<笔记内容>]
   - 探究/思考：source=template_idx 或 duplicate idx=5/12/15/18，placeholders={"12": <小标签>}, add_text_boxes=[<内容>]
5. 头尾固定：
   - P1 封面（template_idx=1）
   - P2 目录（template_idx=2）
   - P3 知识阶梯（add_layout="标题-文字少" + 图片）
   - P4 学习目标（duplicate from_idx=2）
   - 最末几张：附录 4 张（教师备课参考）+ 结束页（template_idx=23）
6. 注意各 placeholder 索引规则（详见 known_template_layouts.md）
```
