# 活动类型 → PPT 版式 映射规则

## 主映射表（规则部分，确定性）

| 教学设计稿 activity 类型 | 主版式 | 默认页数 | slide_role |
|---|---|---|---|
| `情境创设` / `故事推进` | `内文2` | 1 | `story` |
| `探究活动` / `思考` | `内文2` | 1-3（按子段数） | `thinking` |
| `技能学习` / `核心概念讲解` | `内文5` 或 `内文2` | 1-2 | `concept` |
| `例题解析` / `例题讲解` | `内文1` | 1-2（题干+解析） | `example` |
| `学生练习` / `练习` | `1_内文3` | 1 | `practice` |
| `方法总结` / `方法迁移` / `笔记` | `空白页1` | 1-2 | `notes` |
| `模块封面` | `空白页5` | 1 | `module_cover` |
| `时间分配总结` / `故事线回顾` / `核心知识点梳理` | `内文4` 或 `内文5` | 1 | `summary` |

## 必须出现的特殊页（固定结构）

| 序号 | 版式 | slide_role | 内容来源 |
|---|---|---|---|
| 1 | `封面` | `cover` | 讲义标题 + 讲次 + 季节 + 老师 |
| 2 | `目录` | `toc` | 全学期讲义目录（如已注册） |
| 3-4 | `标题-文字多` | `objectives` | 讲义"课程目标"+"知识阶梯" |
| 倒数2 | `内文4` | `lesson_summary` | 教学设计稿"总结与拓展" |
| 倒数1 | `内文5` 或硬编码 | `homework` | 课后任务（讲义"创新挑战" + 教参） |
| 末页 | `结束页` | `end` | "下节课再见！" |

## LLM 在边缘情况上的判断

以下情况需要 LLM 判断（不是纯规则）：

### 1. 一个 activity 展开成几页

```
教学设计稿："探究活动（8分钟）：
1. 数字侦探游戏..."
2. 质因数定义探究..."

→ 这是 1 个 activity 还是 2 个 sub-activity？
→ LLM 判断：通常每个 numbered bullet 是一张 PPT
```

### 2. 复合 activity 的拆分

```
教学设计稿："技能学习（12分钟）：
1. 短除法教学..."
2. 学生练习（5分钟）..."
3. 挑战升级（8分钟）..."

→ 表面上 1 个 activity，但内部含 3 个子段
→ LLM 应该拆成 3 个 PPT 页（技能讲解、学生练习、挑战升级），不要塞到 1 页
```

### 3. 思考题的层级

```
教学设计稿："1. 末尾0的来源..."
        "2. 例题4..."
        "3. 方法总结..."

→ 这是 1 个连续展开还是 3 个独立思考？
→ 看是否有题目编号：例题4 必须独立成例题页
→ 没有编号的 1, 3 → 故事/思考页
```

## 防御性规则

1. **单 activity ≤ 3 页**：防止 LLM 展开过度
2. **总页数 30-40 页**（一节课 PPT 的合理量）
3. **必须包含**：封面 / 目录 / 课程目标 / 至少 1 个例题页 / 结束页
4. **题号严格匹配**：教学设计稿引用"例 N"必须能在讲义里找到 example_num=N
5. **故事页文字 ≤ 80 字**：超过就拆页或精简
